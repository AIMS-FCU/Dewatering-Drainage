# 最佳化一期操作手冊

對應程式：`最佳化正式版PINN一期.py`

## 1. 程式用途

此程式使用一期 PINN 產出的物理參數與背景水位，求解 7 天抽水井開關排程。目標是在滿足目標水位限制下，降低耗電量與開井數。

## 2. 必要輸入檔

| 檔案或資料夾 | 用途 |
|---|---|
| `Master_Training_Data_Continuous3.csv` | 現場實際水位、流量、電力資料 |
| `Distance_Matrix.csv` | 一期距離矩陣 |
| `pinn_report_path` 指向的 PINN 報告資料夾 | 提供 PINN 產物 |

PINN 報告資料夾至少需要：

```text
learned_T.npy
learned_C.npy
calibrated_inflow_sy.npy
background_h_7d.npy 或 accurate_pred_h_7d.npy
```

## 3. 重要設定 `opt_config`

| 參數 | 意義 | 調整建議 |
|---|---|---|
| `ANALYSIS_START/END` | 最佳化分析區間 | 必須與 PINN 預測區間一致或可對齊 |
| `SIM_HOURS` / `SIM_STEPS` | 模擬長度，目前 168 小時 | 7 天固定用 168 |
| `delta_t` | 時間步長，目前 1 小時 | 與排程解析度一致 |
| `pinn_report_path` | PINN 報告資料夾 | 執行前最常需要修改 |
| `WELL_LIST` | 參與最佳化的抽水井 | 必須與距離矩陣、資料欄位一致 |
| `OBS_LIST` | 控制水位的觀測井 | 必須與 PINN 輸出順序可對齊 |
| `FIXED_TARGET_H` | 固定目標水位 | 越淺通常越省電，但風險較高 |
| `SAFETY_BUFFER` | 安全緩衝 | 保守時增加 |
| `MIN_ACTIVE_WELLS` | 每時段最少開井數 | 現場要求最低開井才調 |
| `PRECALIBRATION_ENABLED` | 背景水位預校準 | 通常保持 True |
| `PRECALIBRATION_HOURS` | 用前幾小時校準背景水位 | 常用 24 |

## 4. 核心流程

```text
讀取 PINN 報告資料夾
→ 載入 learned_T / learned_C / calibrated_inflow_sy
→ 讀取背景水位預測
→ 使用現場前段資料做 DynamicCalibrator
→ 建立抽水影響矩陣
→ 使用 PuLP 求解開關排程
→ 比對實際排程與最佳化排程
→ 輸出報告圖與 CSV
```

## 5. 重要函式

| 函式 | 用途 |
|---|---|
| `resolve_pinn_report_path` | 找出可用 PINN 報告資料夾 |
| `align_observation_predictions` | 對齊 PINN 輸出與觀測井順序 |
| `build_actual_obs_matrix` | 建立實際觀測井水位矩陣 |
| `build_influence_matrix` | 建立抽水井對觀測井影響矩陣 |
| `apply_background_precalibration` | 用前段實測資料校準背景水位 |
| `DynamicCalibrator` | 動態校正 Qin 與 T factor |
| `run_pulp_optimization` | 用 PuLP 求解最佳化排程 |
| `compute_carryover_constraints` | 處理起始延續開機限制 |

## 6. 執行方式

```

改成實際 PINN 報告資料夾後執行：

```

## 7. 主要輸出

輸出位置：`pinn_report_path` 指到的資料夾。

| 檔案 | 用途 |
|---|---|
| `Optimization_Report.png` | 最佳化總報告圖 |
| `Well_Open_Comparison.csv` | 每時段實際開井與最佳化開井比對 |
| `Reverse_Verification_Plot.png` | 背景水位/反推驗證圖 |

## 8. 結果判讀

重點看終端輸出與 `Optimization_Report.png`：

- 節能率 `saving_pct`：正值代表最佳化比現場省電。
- 流量偏差 `vol_diff_pct`：絕對值過大代表抽水量偏離現場太多。
- 平均井數：最佳化平均開井數應合理降低。
- 水位 MAE：若大於 0.5 m，需檢查 PINN 背景水位與校準係數。
- 排程熱圖：若出現頻繁開關，需檢查最小開機時間或 carryover 設定。

## 9. 常見調整

- 水位太高，可能不安全：降低 `FIXED_TARGET_H` 或增加 `SAFETY_BUFFER`。
- 太保守、省電不明顯：提高 `FIXED_TARGET_H` 或降低 `SAFETY_BUFFER`。
- 排程過度跳動：增加最小開機時間相關限制。
- PINN 參數不合理：回頭檢查 `learned_T.npy`、`learned_C.npy`、`calibrated_inflow_sy.npy`。
