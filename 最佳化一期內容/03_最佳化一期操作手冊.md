# 最佳化一期操作手冊

本手冊對應程式：

```text
github/最佳化一期內容/最佳化正式版PINN一期.py
```

此程式會讀取一期 PINN 的輸出參數與 7 天預測結果，建立抽水排程最佳化模型，產生節能排程、井開關比較與最佳化報告圖。

## 執行前準備

請確認同一資料夾內至少有：

```text
最佳化正式版PINN一期.py
Master_Training_Data_Continuous3.csv
Distance_Matrix.csv
```

另需準備一期 PINN 結果資料夾。該資料夾至少要有：

```text
learned_T.npy
learned_C.npy
calibrated_inflow_sy.npy
background_h_7d.npy
accurate_pred_h_7d.npy
```

## 執行方式

```powershell
cd github\最佳化一期內容
python 最佳化正式版PINN一期.py
```

執行前務必打開程式，確認：

```python
"pinn_report_path": "PINN_MAPE_Complete_Report3/428_10"
```

這個路徑要指向實際存在的 PINN 結果資料夾。若路徑是相對路徑，會以目前執行資料夾為基準。

## 重要設定 `opt_config`

| 參數 | 說明 | 常見調整 |
|---|---|---|
| `ANALYSIS_START`、`ANALYSIS_END` | 要最佳化的分析期間 | 換排程日期時必改 |
| `SIM_HOURS`、`SIM_STEPS` | 模擬長度與步數，預設 168 小時 | 7 天最佳化通常不動 |
| `delta_t` | 最佳化時間步，小時 | 預設 1 小時 |
| `pinn_report_path` | PINN 結果資料夾 | 執行前最常要改 |
| `WELL_LIST` | 可控制的抽水井 | 必須與資料欄位、距離矩陣一致 |
| `OBS_LIST` | 觀測井水位控制點 | 必須與 PINN 輸出順序一致 |
| `FIXED_TARGET_H` | 目標水位上限 | 要更保守就設更低水位 |
| `SAFETY_BUFFER` | 安全緩衝 | 水位限制太緊或太鬆時調整 |
| `MIN_ACTIVE_WELLS` | 每個時間步至少開幾口井 | 有現場運轉需求時設定 |
| `INITIAL_CARRYOVER_HOURS` | 起始延續狀態時數 | 避免排程一開始大幅跳變 |
| `BUFFER_HOURS` | 抽水切換緩衝 | 排程太頻繁切換時加大 |
| `PRECALIBRATION_ENABLED` | 是否使用前段資料做背景校準 | 建議保持 True |
| `PRECALIBRATION_HOURS` | 校準使用時數 | 預設 24 |

## 程式流程

1. 讀取 `Master_Training_Data_Continuous3.csv`。
2. 讀取 `Distance_Matrix.csv`。
3. 透過 `resolve_pinn_report_path` 找到可用的 PINN 結果資料夾。
4. 載入 PINN 產物：`learned_T.npy`、`learned_C.npy`、`calibrated_inflow_sy.npy`、背景水位與實際條件預測。
5. 建立抽水井到觀測井的影響矩陣。
6. 使用 `DynamicCalibrator` 自動校準 Qin factor 與 T factor。
7. 以 PuLP 建立最佳化問題，目標是降低用電，同時維持水位限制。
8. 輸出最佳化排程與圖表。

## 主要函式

| 函式 | 用途 |
|---|---|
| `resolve_pinn_report_path` | 找到含必要 `.npy` 檔的 PINN 結果資料夾 |
| `build_actual_obs_matrix` | 建立實際觀測井水位矩陣 |
| `build_influence_matrix` | 建立抽水井對觀測井的影響矩陣 |
| `apply_background_precalibration` | 用前段實測資料校正背景水位 |
| `DynamicCalibrator` | 自動調整 Qin 與 T factor |
| `run_pulp_optimization` | 執行 PuLP 最佳化 |
| `compute_carryover_constraints` | 建立開關延續限制 |

## 輸出

輸出會寫到 `pinn_report_path` 指向的 PINN 結果資料夾。

| 檔案 | 說明 |
|---|---|
| `Optimization_Report.png` | 最佳化總報告圖，包含水位、開井數、流量、節能摘要 |
| `Well_Open_Comparison.csv` | 每口井、每個時間步的實際與最佳化開關比較 |
| `Reverse_Verification_Plot.png` | 反向疊加驗證圖，用於檢查背景水位與抽水影響是否閉合 |

## 判讀方式

先看 `Optimization_Report.png`：

- 水位曲線不應高於目標水位或安全界線。
- Optimized Wells 應比 Actual Wells 更少或更平滑，否則節能效果有限。
- Optimized Pumping 應在滿足水位限制下低於實際抽水。
- 若水位 MAE 很大，表示 PINN 背景水位或影響矩陣可能不可靠。

再看 `Well_Open_Comparison.csv`：

- `Actual_*` 是實際開關。
- `Optimized_*` 是建議開關。
- 可用來轉成現場排程表。

## 常見調整

| 現象 | 建議 |
|---|---|
| 最佳化不可行 | 放寬 `FIXED_TARGET_H` 或降低 `SAFETY_BUFFER` |
| 排程開關太頻繁 | 增加 `BUFFER_HOURS` 或起始 carryover 約束 |
| 節能率太低 | 檢查實際抽水是否本來已接近最省，或調整目標水位 |
| 水位超出限制 | 降低 `FIXED_TARGET_H`、增加 `SAFETY_BUFFER`，或檢查 `learned_T.npy` |
| 找不到 PINN 結果 | 確認 `pinn_report_path` 與必要 `.npy` 檔案 |
| 圖中文字亂碼 | Windows 可用 `Microsoft JhengHei`，Linux 需安裝中文字型 |
