# 常見錯誤與除錯

## 1. GPU OOM

錯誤類似：

```text
OOM when allocating tensor with shape [512,4,336,336]
```

原因：Transformer attention 記憶體需求與 `batch_size * num_heads * window_size^2` 相關。

處理：

```python
"batch_size": 512 → 256 → 128 → 64
```

若仍 OOM，減少 `num_transformer_layers` 或 `d_model`。

## 2. XLA 編譯錯誤

錯誤類似：

```text
Operand is not divisible by the block size
```

處理：

```python
ENABLE_XLA = False
tf.config.optimizer.set_jit(ENABLE_XLA)
```

## 3. `KeyError: l_dat`

原因：Keras 版本或自訂 `train_step` 導致 `history.history` 沒有記錄 `l_dat`。

處理：畫 loss 圖時先檢查 key 是否存在；缺少就跳過，不影響模型訓練結果。

## 4. 初始水位偏差 2 m 以上

可能原因：

- scaler fit window 與預測週水位分布不同。
- 遷移學習權重造成水位基準偏移。
- 近期抽水型態與訓練資料不同。

處理：

- 比較 `background_h_7d_raw.npy` 與 `background_h_7d.npy`。
- 查看 `start_level_bias_correction.csv`。
- 報告同時列 Raw MAE 與 Corrected MAE。

## 5. 最佳化找不到 PINN 報告資料夾

錯誤：找不到 `learned_T.npy`、`learned_C.npy`、`calibrated_inflow_sy.npy`。

處理：

1. 確認 `pinn_report_path` 正確。
2. 確認 PINN 程式已完整跑完。
3. 確認報告資料夾內有三個必要 `.npy`。

## 6. 最佳化水位不合理

檢查順序：

1. `background_h_7d.npy` 是否合理。
2. `accurate_pred_h_7d.npy` 是否與實際水位接近。
3. `learned_T.npy`、`learned_C.npy` 是否異常。
4. `Q_factor`、`T_factor` 是否偏離 1 太多。
5. `WELL_LIST` 與距離矩陣欄位是否一致。

## 7. 中文圖表亂碼

一期使用 `Microsoft JhengHei`。二期會優先使用：

```text
SourceHanSansTC-Regular.otf
```

若 Linux 環境亂碼，將字型檔放在專案根目錄，或改成系統已安裝中文字型。

## 8. 結果時好時壞

PINN 與 7 天自回歸預測有天然方差。評估時不要只看單次：

- 固定 seed。
- 比較多個預測週。
- 同時看 Raw MAE、Corrected MAE、Applied Bias。
- 與 baseline 比較，例如維持起始水位不變或前 7 天趨勢外推。
