# 二期新資料與距離矩陣更新說明

本說明對應二期流程。若現場有新的二期資料進來，通常需要同時處理兩件事：

1. 重新產生二期訓練資料：`Phase2_Training_Data2.csv`
2. 確認或重新產生二期距離矩陣：`Distance_Matrix_Phase2.csv`

距離矩陣產生程式為：

```text
generate_mock_distance_matrix.py
```

這支程式是二期用的，輸出檔名是：

```text
Distance_Matrix_Phase2.csv
```

不是一期的 `Distance_Matrix.csv`。

## 新資料進來的標準流程

1. 將新的感測資料整理成 30 分鐘對齊資料。

   預設檔名：

   ```text
   sensor_data_30min_aligned.csv
   ```

2. 執行二期前處理。

   ```powershell
   cd github\PINN二期內容
   python data_preprocess_phase2.py
   ```

   產物：

   ```text
   Phase2_Training_Data2.csv
   ```

3. 若井位沒有變，只要沿用既有 `Distance_Matrix_Phase2.csv`。

4. 若新增井、刪除井、井名改變或有新的實測座標，修改並執行：

   ```powershell
   python generate_mock_distance_matrix.py
   ```

   產物：

   ```text
   Distance_Matrix_Phase2.csv
   ```

5. 確認 `Phase2_Training_Data2.csv`、`Distance_Matrix_Phase2.csv` 和 PINN 程式中的井名一致。

6. 執行二期微調或最佳化。

## `generate_mock_distance_matrix.py` 要改什麼

### 1. 修改抽水井清單

```python
pumping_wells = ["PW01", "PW05", "PW06", "PW07", "PW08", "PW09", "PW010", "PW011"]
```

這裡必須填「會被最佳化控制的抽水井」。清單順序會成為距離矩陣的欄位順序，也會影響 PINN 和最佳化讀取矩陣時的對齊。

若新增一口抽水井 `PW012`，例如：

```python
pumping_wells = ["PW01", "PW05", "PW06", "PW07", "PW08", "PW09", "PW010", "PW011", "PW012"]
```

### 2. 修改觀測井清單

```python
obs_wells = ["PW02", "PW03", "PW04"]
```

這裡填「用來檢查水位限制與模型輸出的觀測井」。二期預設是 `PW02/PW03/PW04`。

若新增觀測井 `PW012`，但它不是抽水井，則加到 `obs_wells`。

### 3. 修改井位座標

```python
coords = {
    "PW01": (96, 30),
    "PW05": (0, 30),
    ...
    "PW02": (72, 15),
}
```

每個 `pumping_wells` 和 `obs_wells` 裡的井都必須在 `coords` 裡有座標。

座標單位建議使用公尺。若有實測平面座標，請直接使用實測值；目前程式內的座標是 mock 示意值，只適合測試流程，不適合正式工程判斷。

### 4. 修改輸出檔名

二期請維持：

```python
output_file = Path("Distance_Matrix_Phase2.csv")
```

只有在你要另存測試版本時才改，例如：

```python
output_file = Path("Distance_Matrix_Phase2_test.csv")
```

但正式跑二期 PINN 或二期最佳化前，程式預設讀的仍是 `Distance_Matrix_Phase2.csv`。

## 距離矩陣格式

輸出的 CSV 會長這樣：

```text
,PW01,PW05,PW06,PW07,PW08,PW09,PW010,PW011
PW02,...
PW03,...
PW04,...
PW01,...
PW05,...
...
```

格式規則：

- columns 是抽水井：`pumping_wells`
- index 是觀測井加抽水井：`obs_wells + pumping_wells`
- 自己到自己的距離是 `0`
- 其他距離用座標的歐氏距離計算

PINN 二期程式會讀：

```python
dist_df = pd.read_csv("Distance_Matrix_Phase2.csv", index_col=0)
```

並取出：

```python
dist_df.loc[obs_cols + wells_list, wells_list]
```

所以 index 和 columns 的井名必須完全對得上。

## 新資料欄位要注意什麼

二期前處理與後續模型通常會用到下列欄位：

| 類型 | 欄位範例 | 注意事項 |
|---|---|---|
| 觀測井水位 | `PW02`, `PW03`, `PW04` | 要和 `obs_wells` 對應 |
| 抽水井水位 | `PW01`, `PW010`, `PW011`, `PW05` | 要和抽水井名一致 |
| 抽水流量 | `QW01`, `QW010` | `010` 不要改成 `10` |
| 即時功率 | `KW01`, `KW010` | 二期最佳化會用來估算各井功率 |
| 累積電表 | `KWH01`, `KWH010` | 前處理會用 `.diff()` 推算 `P_gr_**` 和 `X_**` |
| 開關狀態 | `X_01`, `X_010` | 由前處理產生，不需要原始資料先提供 |
| 推估功率 | `P_gr_01`, `P_gr_010` | 由前處理產生 |

井號規則非常重要：

- `PW010` 對應 `QW010`、`KW010`、`KWH010`、`X_010`、`P_gr_010`
- `PW01` 對應 `QW01`、`KW01`、`KWH01`、`X_01`、`P_gr_01`

不要把 `010` 手動簡化成 `10`，否則資料、距離矩陣與模型會對不起來。

## 新資料前處理會做什麼

`data_preprocess_phase2.py` 會：

- 讀取 `sensor_data_30min_aligned.csv`
- 將 index 轉成時間
- 將欄位轉成數值
- 對 `PW*` 水位欄位做異常值處理：大於 0 或小於 -30 設為 NaN，再插值補齊
- 從 `KWH**` 推出 `P_gr_**`
- 從 `KWH**` 增量推出 `X_**`
- 如果 `X_** = 0`，把對應 `QW**` 設為 0
- 將雨量缺值補 0
- 其餘欄位以前補、後補和 0 補齊
- 輸出 `Phase2_Training_Data2.csv`

## 執行後檢查

檢查訓練資料：

```powershell
python -c "import pandas as pd; df=pd.read_csv('Phase2_Training_Data2.csv', index_col=0); print(df.shape); print(df.head()); print(df.isna().sum().sort_values(ascending=False).head(20))"
```

檢查距離矩陣：

```powershell
python -c "import pandas as pd; d=pd.read_csv('Distance_Matrix_Phase2.csv', index_col=0); print(d.shape); print(d.index.tolist()); print(d.columns.tolist()); print(d.head())"
```

確認重點：

- `Distance_Matrix_Phase2.csv` columns 必須等於抽水井清單。
- `Distance_Matrix_Phase2.csv` index 必須包含觀測井與抽水井。
- `Phase2_Training_Data2.csv` 必須有 `obs_wells` 中的水位欄位。
- `Phase2_Training_Data2.csv` 必須有每口抽水井對應的 `QW/KW/KWH` 欄位，至少要能推得抽水狀態。
- 若有新增井，PINN 二期程式與最佳化二期程式中的 `WELL_LIST`、`OBS_LIST` 也要一起改。

## 哪些程式也要同步修改

若井位或井名變動，至少檢查這些位置：

| 程式 | 要檢查的地方 |
|---|---|
| `generate_mock_distance_matrix.py` | `pumping_wells`、`obs_wells`、`coords` |
| `data_preprocess_phase2.py` | `input_file`、`output_file`，以及資料欄位是否符合 `QW/KW/KWH/PW` 命名 |
| `PINN正式_Phase2_Training_Transfer_二期微調.py` | 二期觀測井與抽水井自動偵測結果、`training_data_path`、task 日期 |
| `最佳化正式版PINN二期.py` | `WELL_LIST`、`OBS_LIST`、`pinn_report_path`、分析日期 |

## 什麼時候不用重產距離矩陣

只有資料時間延長，例如多了新的月份，但井位、井名和座標完全沒變，就不用重跑 `generate_mock_distance_matrix.py`。這種情況只需要重跑：

```powershell
python data_preprocess_phase2.py
```

然後再重新微調 PINN 或執行最佳化。

## 什麼時候一定要重產距離矩陣

- 新增抽水井。
- 新增觀測井。
- 移除井。
- 井名改變。
- 取得更準確的實測座標。
- 原本只是 mock 座標，現在要做正式分析。

正式分析時，請優先使用實測井位座標，不要依賴 mock 座標。
