# 資料前處理手冊

本手冊說明如何把現場原始資料整理成 PINN 訓練需要的 CSV。此資料夾共有兩個前處理程式：一期使用 `data_preporcess.py`，二期使用 `data_preprocess_phase2.py`。

## 前處理目標

PINN 訓練需要每個時間點都有完整的水位、抽水流量、電力與開關狀態欄位。前處理程式會負責：

- 讀取不同來源的 CSV。
- 自動偵測編碼與標題列。
- 將時間欄位轉成 pandas datetime index。
- 將資料重採樣為 30 分鐘間距。
- 從累積電表 `KWH` 推估半小時功率 `P_gr_**`。
- 從電表變化建立抽水開關狀態 `X_**`。
- 在馬達未運轉時將對應抽水流量設為 0。
- 將缺值以前後補值或 0 補齊。

## 一期前處理

### 使用檔案

位置：

```text
github/PINN一期內容/data_preporcess.py
```

執行方式：

```powershell
cd github\PINN一期內容
python data_preporcess.py
```

### 原始資料需求

程式內的 `files` 字典定義了要讀取的原始資料檔。檔名需放在同一個資料夾，或把路徑改成正確位置。

| key | 內容 | 說明 |
|---|---|---|
| `well_drawdown` | 抽水井水位 | 抽水井水位欄位，通常對應 `PW**` |
| `obs_level` | 觀測井水位 | 觀測井水位欄位，例如 `PA`、`PB`、`FPS*` |
| `rain` | 雨量 | 雨量欄位，缺值會補 0 |
| `power` | 電力或電表 | 包含 `KWH**` 等累積電量欄位 |
| `flow` | 抽水流量 | 包含 `Qw**` 或類似抽水量欄位 |

若現場檔名不同，請直接修改：

```python
files = {
    "well_drawdown": "...csv",
    "obs_level": "...csv",
    "rain": "...csv",
    "power": "...csv",
    "flow": "...csv",
}
```

### 欄位產生規則

| 欄位 | 來源或算法 | 用途 |
|---|---|---|
| 水位欄位 | 原始水位 CSV | PINN 水位預測目標與輸入特徵 |
| `Qw**` | 原始抽水流量 | PINN 流量預測與最佳化基準 |
| `KWH**` | 原始累積電表 | 推算功率與開關狀態 |
| `P_gr_**` | `KWH.diff().clip(lower=0) * 2` | 半小時資料換算成 kW 估計值 |
| `X_**` | `KWH` 半小時增量是否大於門檻 | 井是否正在抽水 |
| 雨量欄位 | 原始雨量 | 外生輸入特徵 |

注意：`X_** = 0` 時，程式會把對應 `Qw**` 設為 0，避免停機時因感測器殘值讓模型誤判仍在抽水。

### 原始資料需求

程式預設讀取：

```text
sensor_data_30min_aligned.csv
```

此檔應已經是 30 分鐘對齊資料。若檔名不同，修改程式開頭：

```python
input_file = "sensor_data_30min_aligned.csv"
output_file = "Phase2_Training_Data2.csv"
```

### 二期欄位規則

| 欄位 | 說明 |
|---|---|
| `PW02`、`PW03`、`PW04` | 二期觀測井水位，通常也是最佳化控制目標 |
| `PW01`、`PW010`、`PW011`、`PW05`、`PW06`、`PW07`、`PW08`、`PW09` | 二期抽水井水位或相關井位 |
| `QW**` / `Qw**` | 二期抽水流量 |
| `KW**` | 即時功率，若存在會供最佳化估算井功率 |
| `KWH**` | 累積電量，可推估功率與開關狀態 |
| `P_gr_**` | 由 `KWH` 推算的半小時功率 |
| `X_**` | 由電表變化推算的抽水開關 |

二期井號常見有 `010`、`011` 這種三碼井號，不要手動改成 `10`、`11`，否則 `QW010`、`KWH010`、`X_010` 會對不起來。

## 前處理後檢查

建議每次產出訓練 CSV 後都做以下檢查：

```powershell
python -c "import pandas as pd; df=pd.read_csv('Phase2_Training_Data2.csv', index_col=0); print(df.shape); print(df.head()); print(df.isna().sum().sort_values(ascending=False).head(20))"
```

一期請把檔名換成：

```text
Master_Training_Data_Continuous3.csv
```

檢查重點：

- `shape` 的列數是否符合資料期間。
- index 是否為時間。
- 是否有大量 NaN。
- 抽水流量欄位在停機時是否接近 0。
- `P_gr_**` 不應出現大量負值。
- `X_**` 應為 0/1。
- 觀測井水位不應出現明顯不合理跳點，例如突然跳數十公尺。

## 常見問題

### 找不到原始 CSV

確認目前工作目錄是否在程式所在資料夾。若原始資料放在上一層或其他資料夾，請改成相對路徑：

```python
input_file = "../../data/sensor_data_30min_aligned.csv"
```

### 時間欄位解析失敗

程式會取第一欄當時間欄。若 CSV 第一欄不是時間，請先整理 CSV，或在程式中明確指定時間欄。

### 抽水量全部變 0

通常是井號對不上。請檢查：

- `KWH01` 對應 `Qw01` 或 `QW01`
- `KWH010` 對應 `QW010`
- `X_010` 是否存在

### CSV 開啟亂碼

輸出使用 `utf-8-sig`，Excel 通常可正常開啟。若終端顯示亂碼，可先設定 PowerShell：

```powershell
chcp 65001
$env:PYTHONUTF8=1
```

## 二期新資料與距離矩陣

若二期只是新增時間資料，例如多了一段新的監測月份，但井位與井名完全沒變，通常只需要重新執行：

```powershell
cd github\PINN二期內容
python data_preprocess_phase2.py
```

若新資料包含新增井、刪除井、井名改變或新的井位座標，還要更新二期距離矩陣：

```powershell
python generate_mock_distance_matrix.py
```

此程式是二期用的，輸出為：

```text
Distance_Matrix_Phase2.csv
```

要修改的地方是：

- `pumping_wells`：二期抽水井清單。
- `obs_wells`：二期觀測井清單。
- `coords`：每口井的平面座標，正式分析請使用實測座標。

完整說明請看：

