# 資料前處理操作手冊

本章說明如何把原始感測器資料整理成 PINN 與最佳化程式可直接讀取的訓練資料表。接手者只要先確認本章輸出檔正常，後面的模型訓練與最佳化才有意義。

## 程式與輸出

| 階段 | 程式 | 主要輸入 | 主要輸出 |
|---|---|---|---|
| 一期 | `data_preporcess.py` | 多個原始 CSV，包含水位、觀測井、雨量、電表、流量 | `Master_Training_Data_Continuous3.csv` |
| 二期 | `data_preprocess_phase2.py` | `sensor_data_30min_aligned.csv` | `Phase2_Training_Data2.csv` |

注意：一期程式檔名是 `data_preporcess.py`，preprocess 拼字有誤但目前程式以此檔名存在。除非同步更新所有引用與手冊，否則不要直接改名。

## 一期前處理

執行指令：

```powershell
python data_preporcess.py
```

一期程式會做以下事情：

1. 讀取多個原始 CSV。
2. 自動偵測檔案編碼。
3. 自動尋找時間欄所在的表頭列。
4. 將第一欄轉成時間索引。
5. 移除空欄與無關欄位。
6. 將數值欄位去逗號、去空白並轉成數字。
7. 以 30 分鐘為單位重採樣。
8. 合併水位、觀測井、雨量、電表、流量資料。
9. 由電表累積度數產生抽水機狀態與功率特徵。
10. 當抽水機關閉時，將對應流量歸零。
11. 雨量缺值補 0，其餘缺值以前後資料補齊。
12. 輸出 `Master_Training_Data_Continuous3.csv`。

### 一期重要欄位

| 欄位型態 | 說明 |
|---|---|
| 水位欄 | 抽水井或觀測井水位，供 PINN 預測水位使用 |
| `Qw**` | 抽水流量欄位 |
| `KWH**` | 電表累積用電量 |
| `P_gr_**` | 由 `KWH` 差分換算的半小時功率特徵 |
| `X_**` | 抽水機開關狀態，1 代表有運轉，0 代表停機 |
| rain 相關欄 | 雨量或外部補注特徵 |

`P_gr_**` 與 `X_**` 是後續 PINN 學習抽水機狀態的重要特徵，不只是報表欄位。

## 二期前處理

執行指令：

```powershell
python data_preprocess_phase2.py
```

二期程式假設資料已經先對齊成 30 分鐘一筆，主要輸入是：

```text
sensor_data_30min_aligned.csv
```

二期程式會做以下事情：

1. 讀取 `sensor_data_30min_aligned.csv`。
2. 將所有欄位轉成數值。
3. 找出 `PW` 開頭的水位欄位。
4. 將不合理水位視為異常值：
   - 水位大於 0
   - 水位小於 -30
5. 對異常水位做線性插補，再以前後資料補齊。
6. 找出 `KWH` 電表欄位。
7. 由 `KWH` 差分產生 `P_gr_**` 與 `X_**`。
8. 找出對應的 `QW**` 或 `Qw**` 流量欄。
9. 當 `X_** = 0` 時，將對應流量歸零。
10. 雨量欄位缺值補 0，其餘缺值以前後資料補齊。
11. 輸出 `Phase2_Training_Data2.csv`。

### 二期重要欄位

| 欄位型態 | 說明 |
|---|---|
| `PW**` | 二期抽水井水位 |
| `QW**` / `Qw**` | 二期抽水流量 |
| `KWH**` | 電表累積用電量 |
| `P_gr_**` | 電表差分換算的功率特徵 |
| `X_**` | 抽水機開關狀態 |

二期程式會保留井號中的完整數字，例如 `010` 不會被改成 `10`。這點很重要，因為後續模型會用井號對應流量欄與距離矩陣。

## 前處理後檢查清單

每次前處理完成後，至少檢查以下項目：

1. 輸出 CSV 是否存在。
2. 時間索引是否是 30 分鐘間距。
3. 時間是否由早到晚排序。
4. 是否還有大量 NaN。
5. 水位是否落在合理範圍，例如約 `-30m` 到 `0m`。
6. 抽水機關閉時，對應流量是否為 0。
7. `P_gr_**` 是否沒有大量不合理尖峰。
8. PINN 使用的井名是否存在於距離矩陣。
9. 流量欄位與抽水井欄位是否能一一對應。

可用快速檢查範例：

```powershell
python -c "import pandas as pd; df=pd.read_csv('Phase2_Training_Data2.csv', index_col=0); print(df.shape); print(df.isna().sum().sort_values(ascending=False).head(20)); print(df.head())"
```

一期請把檔名換成：

```text
Master_Training_Data_Continuous3.csv
```

## 與後續程式的關係

| 前處理輸出 | 後續使用程式 |
|---|---|
| `Master_Training_Data_Continuous3.csv` | `PINN正式版本.py`、`最佳化正式版PINN一期.py` |
| `Phase2_Training_Data2.csv` | `PINN正式_Phase2_Training_Transfer_二期微調.py`、`最佳化正式版PINN二期.py` |

如果前處理輸出欄位錯誤，PINN 可能仍會開始訓練，但結果通常會表現為：

- 水位 MAE 異常大。
- 抽水量預測貼近 0 或亂跳。
- 7 天自回歸曲線突然崩掉。
- 最佳化結果看似節能，但水位驗證不合理。

## 常見問題

### 找不到輸入檔

先確認程式內設定的檔名是否與實際檔名一致。一期程式的原始檔名可能因編碼顯示成亂碼，接手者應依照資料類型檢查 `files` 字典，而不是只看檔名表面文字。

### 時間欄解析失敗

確認 CSV 裡是否真的有時間欄，且格式能被 `pandas.to_datetime` 解析。若原始檔前面有說明列，需確認程式是否正確找到表頭列。

### 流量全部變 0

通常是 `KWH` 欄位名稱與 `QW/Qw` 欄位名稱對不起來，導致開關狀態沒有正確套用。請檢查井號後綴，例如 `01`、`010`、`011` 是否一致。

### 水位被補得太平

代表原始資料缺值或異常值太多。需回頭確認感測器資料品質，不建議只靠插補硬補出漂亮曲線。

### 距離矩陣對不到井名

PINN 訓練前會用水位欄與距離矩陣交集。如果前處理輸出的井名和 `Distance_Matrix*.csv` 不一致，模型會少井或直接報錯。
