# 最佳化二期操作手冊

本手冊對應程式：

```text
github/最佳化二期內容/最佳化正式版PINN二期.py
```

此程式會使用二期 PINN 微調結果，針對二期抽水井建立 7 天抽水排程最佳化。相較一期，二期版本多了目標水位 quantile 模式、井功率估算、過抽懲罰、自動放寬目標與 local search 等功能。

## 執行前準備

請確認同一資料夾內至少有：

```text
最佳化正式版PINN二期.py
Phase2_Training_Data2.csv
Distance_Matrix_Phase2.csv
```

若要避免圖表中文字亂碼，建議也放：

```text
SourceHanSansTC-Regular.otf
```

另需準備二期 PINN 結果資料夾，至少包含：

```text
learned_T.npy
learned_C.npy
calibrated_inflow_sy.npy
background_h_7d.npy
accurate_pred_h_7d.npy
```

## 執行方式

```powershell
cd github\最佳化二期內容
python 最佳化正式版PINN二期.py
```

執行前務必確認：

```python
"pinn_report_path": "PINN_Phase2_20260301_Test"
```

這個路徑要改成實際的二期 PINN 輸出資料夾，例如：

```python
"pinn_report_path": "../PINN二期內容/微調/Report_Full_0301_Early"
```

## 二期井位設定

預設抽水井：

```text
PW01, PW010, PW011, PW05, PW06, PW07, PW08, PW09
```

預設觀測井：

```text
PW02, PW03, PW04
```

`WELL_LIST`、`OBS_LIST`、`Distance_Matrix_Phase2.csv`、`Phase2_Training_Data2.csv` 欄位必須一致。

## 重要設定 `opt_config`

| 參數 | 預設值 | 說明 |
|---|---:|---|
| `ANALYSIS_START`、`ANALYSIS_END` | 2026-03-01 到 2026-03-08 | 最佳化分析期間 |
| `AREA` | 5232 | 二期場域面積 |
| `delta_t` | 1.0 | 最佳化時間步，小時 |
| `SIM_HOURS`、`SIM_STEPS` | 168 | 7 天、每小時一步 |
| `pinn_report_path` | `PINN_Phase2_20260301_Test` | 二期 PINN 結果資料夾 |
| `TARGET_MODE` | `quantile` | 目標水位模式，可用 `quantile` 或 `fixed` |
| `TARGET_QUANTILE` | 0.95 | 使用歷史實測水位分位數當目標 |
| `FIXED_TARGET_H` | -20.6 | `TARGET_MODE="fixed"` 時使用 |
| `TARGET_UPPER_TOLERANCE_M` | 0.3 | 允許高於目標的容忍範圍 |
| `TARGET_LOWER_TOLERANCE_M` | 1.2 | 低於目標過多視為過抽 |
| `OVER_PUMPING_PENALTY` | 10000 | 過抽懲罰 |
| `TARGET_TRACKING_PENALTY` | 300 | 追蹤目標水位懲罰 |
| `AUTO_RELAX_FOR_WELL_SAVING` | True | 若省井效果不足，自動放寬目標 |
| `MIN_UP_TIME` | 4 | 最短連續運轉時間 |
| `PRECALIBRATION_ENABLED` | True | 是否做背景水位前校準 |

## 目標水位模式

### quantile 模式

預設模式：

```python
"TARGET_MODE": "quantile",
"TARGET_QUANTILE": 0.95,
```

程式會從分析期間的實測觀測井水位取分位數當目標。因地下水位是負值，較高分位數通常代表比較淺的水位，會比較省電，但也要確認仍符合安全需求。

### fixed 模式

若現場已有明確水位標準，改用：

```python
"TARGET_MODE": "fixed",
"FIXED_TARGET_H": -20.6,
```

此時所有觀測井會使用固定目標水位。

## 程式流程

1. 讀取 `Phase2_Training_Data2.csv`。
2. 讀取 `Distance_Matrix_Phase2.csv`。
3. 找到 `pinn_report_path` 指向的二期 PINN 結果。
4. 載入 `learned_T.npy`、`learned_C.npy`、`calibrated_inflow_sy.npy`、`background_h_7d.npy`、`accurate_pred_h_7d.npy`。
5. 從 `KW/QW/KWH` 推估每口井的實際功率。
6. 建立目標水位陣列。
7. 透過 `DynamicCalibrator` 校準 Qin 與 T factor。
8. 建立 PuLP 最佳化問題。
9. 若啟用 greedy / local search，進一步降低開井數或用電。
10. 輸出報告圖與 CSV。

## 主要函式

| 函式 | 用途 |
|---|---|
| `estimate_phase2_well_power_kw` | 從 `KW/QW/KWH` 估算每口井功率 |
| `build_target_h_array` | 依 `fixed` 或 `quantile` 建立目標水位 |
| `resolve_pinn_report_path` | 找出可用的 PINN 結果資料夾 |
| `align_observation_predictions` | 對齊 PINN 觀測井預測 |
| `apply_background_precalibration` | 校正背景水位 |
| `DynamicCalibrator` | 自動調整 Qin factor 與 T factor |
| `run_pulp_optimization` | 執行最佳化 |
| `build_well_power_summary` | 建立每口井功率摘要 |

## 輸出

輸出會寫入 `pinn_report_path` 指向的資料夾。

| 檔案 | 說明 |
|---|---|
| `Optimization_Report.png` | 二期最佳化總報告 |
| `Well_Open_Comparison.csv` | 實際與最佳化開關排程比較 |
| `Well_Power_Summary.csv` | 每口井功率估算來源與結果 |
| `Reverse_Verification_Plot.png` | 背景水位、反向推導與 PINN 預測閉合驗證 |

## 判讀方式

### `Optimization_Report.png`

重點看：

- 實際水位、最佳化水位是否滿足目標線。
- 最佳化開井數是否少於實際開井數。
- 最佳化抽水量是否合理，不應為了省電導致水位違規。
- 過抽區間是否減少。
- 節能摘要是否與排程變化一致。

### `Well_Open_Comparison.csv`

可用來檢查每一小時每口井是否開啟。若要交給現場使用，可從這份檔案整理成排程表。

### `Well_Power_Summary.csv`

用來確認每口井功率估算是否合理。若某口井功率是 fallback 值，代表原始資料的 `KW/QW/KWH` 可能不足，節能估算需保守解讀。

## 常見調整

| 現象 | 建議 |
|---|---|
| 最佳化不可行 | 放寬 `TARGET_UPPER_TOLERANCE_M`、降低 `SAFETY_BUFFER`、或改用較保守的 fixed 目標 |
| 開井數沒有下降 | 開啟或加強 `AUTO_RELAX_FOR_WELL_SAVING`，或調整 `TARGET_AVG_WELL_REDUCTION` |
| 水位低太多，疑似過抽 | 提高 `OVER_PUMPING_PENALTY` 或降低 `TARGET_LOWER_TOLERANCE_M` |
| 排程頻繁切換 | 增加 `MIN_UP_TIME` |
| 功率估算不可信 | 檢查 `KW**`、`QW**`、`KWH**` 欄位與井號 |
| 圖表中文字亂碼 | 放入 `SourceHanSansTC-Regular.otf`，或安裝 `Microsoft JhengHei` |
| 找不到 PINN 結果 | 檢查 `pinn_report_path` 與必要 `.npy` 檔 |

## 新資料或新井位進來時

若只是新增二期時間資料，井位與井名沒有變，最佳化通常只需要使用新的 `Phase2_Training_Data2.csv` 和新的二期 PINN 結果資料夾，然後修改：

```python
"pinn_report_path": "新的二期 PINN 結果資料夾"
"ANALYSIS_START": "新的分析起始時間"
"ANALYSIS_END": "新的分析結束時間"
```

若新增井、刪除井、井名改變或取得新的座標，請先回到：

```text
github/PINN二期內容/generate_mock_distance_matrix.py
```

修改：

- `pumping_wells`
- `obs_wells`
- `coords`

並重新產生：

```text
Distance_Matrix_Phase2.csv
```

接著同步檢查本程式的 `WELL_LIST`、`OBS_LIST` 是否和新的距離矩陣、`Phase2_Training_Data2.csv` 欄位一致。完整流程請看 `github/PINN二期內容/00_二期新資料與距離矩陣更新說明.md`。
