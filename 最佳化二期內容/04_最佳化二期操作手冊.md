# 最佳化二期操作手冊

對應程式：`最佳化正式版PINN二期.py`

## 1. 程式用途

此程式使用二期 PINN 微調結果，對二期抽水井進行 7 天最佳化排程。相較一期版本，二期版本加入：

- 二期井名與欄位命名處理。
- 自動估算各抽水井功率。
- 目標水位可用 quantile 自動產生。
- 目標水位上下容忍帶。
- 過抽懲罰與目標追蹤懲罰。
- greedy hard upper 與 local search。
- 實際/最佳化/差異三組熱圖。

## 2. 必要輸入檔

| 檔案或資料夾 | 用途 |
|---|---|
| `Phase2_Training_Data2.csv` | 二期實際資料 |
| `Distance_Matrix_Phase2.csv` | 二期距離矩陣 |
| `pinn_report_path` 指向的 PINN 二期報告資料夾 | 提供 PINN 產物 |
| `SourceHanSansTC-Regular.otf` | 可選，中文字型 |

PINN 報告資料夾至少需要：

```text
learned_T.npy
learned_C.npy
calibrated_inflow_sy.npy
background_h_7d.npy 或 accurate_pred_h_7d.npy
```

## 3. 重要設定 `opt_config`

| 參數 | 目前概念 | 說明 |
|---|---|---|
| `ANALYSIS_START/END` | 分析區間 | 建議與 PINN 預測區間一致 |
| `SIM_HOURS/SIM_STEPS` | 168 | 7 天、每步 1 小時 |
| `pinn_report_path` | PINN 結果資料夾 | 執行前必改 |
| `WELL_LIST` | 二期抽水井 | 預設 8 口 |
| `OBS_LIST` | 二期觀測井 | 預設 `PW02/PW03/PW04` |
| `TARGET_MODE` | `quantile` 或 `fixed` | 目標水位來源 |
| `TARGET_QUANTILE` | 0.95 | quantile 目標，通常較淺、省電 |
| `FIXED_TARGET_H` | 固定目標水位 | `TARGET_MODE=fixed` 時使用 |
| `TARGET_UPPER_TOLERANCE_M` | 高水位容忍 | 不可比目標高太多 |
| `TARGET_LOWER_TOLERANCE_M` | 低水位容忍 | 低太多視為過抽 |
| `OVER_PUMPING_PENALTY` | 過抽懲罰 | 避免水位被抽太低 |
| `TARGET_TRACKING_PENALTY` | 目標追蹤懲罰 | 避免離目標太遠 |
| `AUTO_RELAX_FOR_WELL_SAVING` | 自動放寬目標 | 若省井數不足，目標往淺水位放寬 |
| `MIN_UP_TIME` | 最小開機時間 | 避免頻繁啟停 |
| `PRECALIBRATION_ENABLED` | 背景預校準 | 通常保持 True |

## 4. 目標水位模式

### fixed

使用：

```python
"TARGET_MODE": "fixed"
"FIXED_TARGET_H": -20.6
```

適合已有明確設計水位時。

### quantile

使用分析期間實測觀測井水位自動取分位數：

```python
"TARGET_MODE": "quantile"
"TARGET_QUANTILE": 0.95
```

水位為負值時，較高分位數通常代表比較淺的水位，較省電但安全裕度較少。

## 5. 核心流程

```text
讀取二期 PINN 報告資料夾
→ 載入 learned_T / learned_C / calibrated_inflow_sy
→ 讀取 background_h_7d 或 accurate_pred_h_7d
→ 對齊觀測井順序
→ 從 KW/QW/KWH 欄位估算每口井功率
→ 用 DynamicCalibrator 校準 Q factor / T factor
→ 建立目標水位與容忍帶
→ 求解最佳化排程
→ 若啟用 AUTO_RELAX，必要時放寬目標重跑
→ 輸出排程、功率摘要、圖表
```

## 6. 重要函式

| 函式 | 用途 |
|---|---|
| `estimate_phase2_well_power_kw` | 從 KW/QW/KWH 推估井功率 |
| `build_target_h_array` | 建立 fixed 或 quantile 目標水位 |
| `resolve_pinn_report_path` | 找 PINN 報告資料夾 |
| `align_observation_predictions` | 對齊 PINN 觀測井輸出 |
| `apply_background_precalibration` | 背景水位預校準 |
| `DynamicCalibrator` | 校準 Qin 與 T factor |
| `run_pulp_optimization` | 求解最佳化排程 |
| `build_well_power_summary` | 輸出每口井耗電摘要 |

## 9. 結果判讀

| 現象 | 可能意義 | 處理 |
|---|---|---|
| 節能率高但水位超上限 | 太省電、不安全 | 降低目標水位或增加懲罰 |
| 水位遠低於目標 | 過抽 | 增加 `OVER_PUMPING_PENALTY` 或放寬目標 |
| 流量偏差過大 | 井功率或 kW_to_m3h 轉換不準 | 檢查 KW/QW/KWH 欄位 |
| 排程頻繁跳動 | 限制不足 | 增加 `MIN_UP_TIME` |
| 幾乎沒有省電 | 目標太保守 | 使用 quantile 或啟用 auto relax |
| 水位 MAE > 0.5 m | PINN 背景水位或校準不準 | 回看 PINN 預測圖與 calibration |
